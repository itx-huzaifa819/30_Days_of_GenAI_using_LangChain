## 1. Definition of Prompts

A **prompt** is an input, instruction, or query sent to a model to guide its output.

### Types of Prompts

| Type | Description |
|------|-------------|
| **Text-based Prompts** | Use only text inputs |
| **Multimodal Prompts** | Accept a combination of text, images, audio, video, and documents simultaneously |

> 💡 LLMs act as the prompt-sensitive model for multimodal input.

---

## 2. Prompt Categories

| Category | Description |
|----------|-------------|
| **Static Prompts** | Fixed prompts that do not change |
| **Dynamic Prompts** | Prompts that change based on variables or context |

---

## 3. Prompt Templates

A **prompt template** is a structured way to create prompts dynamically by inserting variables into a predefined template.

### Why Use Prompt Templates?

- ✅ **Default Validation** – Ensures inputs conform to expected formats
- ♻️ **Reusability** – Write once, use across multiple scenarios
- 🔗 **Consistency** – Maintains uniformity within the language ecosystem

---

## 4. Types of Messages in LLM Ecosystem

LLM-based conversations use three core message types:

### 🔧 System Message
- Hidden foundation instructions sent to a model at the start of a conversation
- Defines the model's **persona**, **tone**, **behavior**, and **boundaries**

### 👤 Human Message
- Direct input, query, or prompt provided by the user

### 🤖 AI Message
- Text-based response generated by the AI using NLP and machine learning for human-like conversational output

---

## 5. Chat Prompt Templates

Chat prompt templates support two usage patterns:

| Pattern | Description |
|---------|-------------|
| **Single Message** | Single-turn, standalone queries |
| **List of Messages** | Multi-turn conversation with multiple messages |

### Message Structure Overview

```
Messages
├── Static Messages
│   ├── System Message
│   ├── Human Message
│   └── AI Message
│
└── Dynamic Messages
    └── Chat Prompt Templates (insert variables at runtime)
```

---

## 6. Message Placeholder

A **message placeholder** is a special placeholder used inside a chat prompt template to dynamically insert chat history and a list of messages at runtime.
